พัฒนาเว็บ

นักพัฒนาฝั่งหลัง

สร้างระบบที่ขับเคลื่อนแอปพลิเคชัน: API, ฐานข้อมูล, การยืนยันตัวตน และเซิร์ฟเวอร์ งานฝั่งหลังมองไม่เห็นสำหรับผู้ใช้ — แต่สำคัญต่อทุกสิ่งที่พวกเขาทำ

1

เลือกภาษา

เริ่มด้วย Node.js ถ้าคุณรู้จัก JavaScript อยู่แล้ว, Python (กับ FastAPI) ถ้าชอบอ่านง่าย, หรือ Go ถ้าต้องการประสิทธิภาพ Node.js คือทางเข้าที่เร็วที่สุดสำหรับนักพัฒนาเว็บ

Node.js — เริ่มต้น →
2

Express.js — API แรกของคุณ

สร้าง HTTP server ด้วย Express: routes, middleware, request/response handling และ error handling เขียน REST API ง่ายๆ ครบทั้ง GET, POST, PUT และ DELETE

Express.js — Hello World →
3

ฐานข้อมูล — SQL และ NoSQL

เรียนฐานข้อมูล relational ด้วย PostgreSQL: tables, joins, indexes และ transactions รวมถึงเข้าใจ MongoDB สำหรับข้อมูลแบบ document โปรเจกต์จริงส่วนใหญ่ใช้ทั้งสองแบบ

บทเรียน PostgreSQL →
4

Authentication — JWT และ OAuth

สร้างระบบ login ด้วย JWT token: sign token ตอน login, verify ตอนเข้า route ที่ป้องกัน, จัดการ refresh token และเข้าใจว่าทำไมถึงไม่เก็บรหัสผ่านแบบ plain text เด็ดขาด

jwt.io — บทนำ →
5

การออกแบบ REST API

เรียนหลักการ REST: การตั้งชื่อ resource, status code, versioning, pagination และรูปแบบ error response API ที่ออกแบบดีใช้งานสนุก — ออกแบบแย่เป็นหายนะทางอาชีพ

restfulapi.net →
6

Deploy — Railway หรือ Render

Push server ไปยัง cloud platform Railway และ Render เป็นตัวเลือกที่เป็นมิตรกับมือใหม่ที่สุด เชื่อมต่อฐานข้อมูล ตั้ง environment variables แล้วได้ API URL ที่ใช้งานได้จริง

Railway →

เคล็ดลับ: สร้าง CRUD API ง่ายๆ (tasks, notes หรือบล็อก) ก่อนจะไปต่อระบบที่ซับซ้อน เมื่อเข้าใจการไหลของข้อมูลพื้นฐาน — request → validate → database → respond — ทุกอย่างที่เหลือจะง่ายขึ้นทันที
